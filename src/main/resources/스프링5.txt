2. 필터를 이용한 로그인 체크

    로그인 여부를 체크해야 하는 컨트롤러마다 동일하게 체크하는 로직을 작성하면 같은 코드를 계속 작성해야 한다.
    이를 위해 우리는 필터(Servlet Filter)를 이용하여 처리한다.

    필터는 틀정한 서블릿이나 JSP 등에 도달하는 과정에서 필터링하는 역할을 하는 서블릿 객체이다.

    @WebFilter 어노테이션을 이용하여 특정한 경로에 접근할 때 필터가 동작하도록 설계하고 동일한 로직을 필터로 분리할 수 있다.
    필터는 한 개이상, 여러 개를 적용할 수 있다. (필터 체이닝)

8. HttpSession을 이용할 때 JSESSIONID(서버쿠키, 세션쿠키)
    - 생성 : 자동 생성
    - 유효기간 : 지정 불가
    - 브라우저 보관방식 : 메모리 상에서만 보관 가능
    - 쿠키 사이즈(크기) : 4kb

   사용자 정의 쿠키 : 개발자가 직접 쿠키를 생성해서 사용자 정의 쿠키를 사용할 수 있다.
    - 생성 : 개발자가 직접 생성 -> new Cookie()
    - 유효기간 : 지정 (초 단위)
    - 전송 : 반드시 HttpServletRespone에 AddCookie를 통해 클라이언트(브라우저)에 전송
    - 브라우저 보관방식 : 메모리 상에서만 보관 가능하여 유효기간이 있는 경우 파일로 보관 가능
    - 쿠키 사이즈(크기) : 4kb

    쿠키의 사용 예 : 오늘 하루 이 창 열지 않기, 최근 본 상품 목록, 모바일 자동 로그인

    1. Todo 목록에서 조회했던 Todo 번호(Tno)들을 쿠키를 이용해서 보관

    동작방식의 이해
    (1) 브라우저에서 전송된 쿠키가 있는지 확인
        -> 있다면 해당 쿠키의 값(value) 활용, 없다면 새로운 문자열(값) 생성

    (2) 쿠키의 이름을 'viewTods' 지정

    (3) 문자열 내에 현재 Todo의 문자번호를 문자열로 연결

    (4) '2-3-4' 형태로 연결하고 이미 조회한 번호는 추가하지 않는다.

    (5) 쿠키의 유효시간은 1분으로 지정하고 쿠키를 담아 전송
